# xhs-research v2 代码 Review 报告

## Critical（必须修）

### 1. start.py: 文件句柄泄漏

**文件**: `scripts/start.py:38-44`

```python
log_file = open(log_path, "a")  # 打开但从未关闭
subprocess.Popen(..., stdout=log_file, stderr=log_file, start_new_session=True)
```

**影响**: 父进程泄漏 FD，长期运行会耗尽文件描述符限制。

**修复**: Popen 启动后关闭文件句柄（子进程已继承 FD）。

---

## High（应该修）

### 2. xhs_research.py: "哪个好" 在两个意图分类中重复

**文件**: `scripts/xhs_research.py:50-51`

```python
"推荐": [..., "哪个好", ...],
"测评": [..., "哪个好", ...],
```

**影响**: `classify_query()` 遍历 dict 时「推荐」先匹配，「测评」永远不会被触发。输入「咖啡机哪个好」永远分类为「推荐」。

**修复**: 从「测评」中删除「哪个好」。

### 3. xhs_research.py: _extract_core 全噪声词输入返回原始字符串

**文件**: `scripts/xhs_research.py:89-93`

```python
core = [w for w in words if w not in NOISE_WORDS]
return " ".join(core) if core else topic.strip()  # 全噪声词 → 返回原始
```

**影响**: 输入「请问有没有好的推荐吗」→ 核心词提取失败 → 扩展出「请问有没有好的推荐吗推荐」等无效变体。

**修复**: 全噪声词时返回空字符串，`expand_query_fallback` 中处理空核心的情况。

### 4. SKILL.md: Step 0 路径搜索缺少 .gemini/extensions

**文件**: `SKILL.md:33-41`

SKILL.md 的 bash 路径搜索少了 `$HOME/.gemini/extensions/xhs-research`，但 `_common.py:29` 有。

**修复**: 在 SKILL.md 的 for 循环中加上这个路径。

## Medium（建议修）

### 5. status.py: --json 模式退出码始终为 0

**文件**: `scripts/status.py:41-43`

```python
if emit_json:
    print(json.dumps(results, indent=2))
    return  # 隐式退出码 0
```

**影响**: SKILL.md 中 `status.py --json` 的退出码无法反映真实状态，依赖退出码的自动化会失效。

**修复**: return 前加 `sys.exit(0 if results["all_ready"] else 1)`。

### 6. xhs_research.py: --quick 和 --deep 可以同时传

**文件**: `scripts/xhs_research.py:587`

```python
depth = "quick" if args.quick else "deep"  # --quick 静默赢
```

**影响**: `python3 xhs_research.py "topic" --quick --deep` 不报错但行为不直观。

**修复**: 用 `argparse.add_mutually_exclusive_group()`。

### 7. xhs_research.py: fetch_details 评论数写死 deep 配置

**文件**: `scripts/xhs_research.py:462`

```python
for c in comments_data[:DEPTH_CONFIG["deep"]["comment_top"]]:
```

**影响**: quick 模式下也获取 8 条评论（应该是 5 条）。

**修复**: 传入 depth 参数，用 `DEPTH_CONFIG[depth]["comment_top"]`。

### 8. xhs_research.py: "避雷" fallback 后缀第一个就是 "避雷"

**文件**: `scripts/xhs_research.py:80`

```python
"避雷": ["避雷", "踩坑", "不推荐", "推荐"],
```

**影响**: 输入「咖啡机避雷」→ 变体 2 是「咖啡机避雷」（和原始重复）。虽然 dedup 能过滤，但浪费一个搜索槽。

**修复**: 「避雷」后缀列表改为 `["踩坑", "不推荐", "推荐", "吐槽"]`。

## Low（可选修）

### 9. xhs_research.py: _http_post_json 对所有错误重试

**文件**: `scripts/xhs_research.py:127-132`

404 错误不应该重试（资源不存在，重试无意义）。当前实现对所有 Exception 重试。

**建议**: 区分 urllib.error.HTTPError 的 status code，4xx 不重试。

### 10. start.py: 端口竞争条件

**文件**: `scripts/start.py:19-40`

健康检查和启动之间有时间窗口，另一个进程可能抢占端口。

**影响**: 极低风险（CLI 单线程场景），且已被异常处理覆盖。

### 11. xhs_research.py: render_output 中 remaining 列表取 30 条

**文件**: `scripts/xhs_research.py:531`

```python
for i, item in enumerate(remaining[:30]):
```

硬编码 30，没有参数化。对大多数场景够用。

## Documentation

### 12. SKILL.md: 缺少「避雷」和「攻略」类关键字生成示例

Step 1 有推荐/调研/竞品示例，但没有避雷和攻略类示例。

### 13. README.md: 系统支持描述不一致

常见问题说「macOS 和 Linux」，但没提 Windows (WSL)。系统要求段有提，但不明显。

### 14. login.py: import start 模块路径依赖

**文件**: `scripts/login.py:51`

```python
import start
start.main()
```

依赖 Python 的模块搜索路径（cwd 或 sys.path）。当前 `sys.path.insert(0, ...)` 在文件头部覆盖了，但如果从其他目录调用可能出问题。

**建议**: 改为显式 `from start import main as start_main`。
